# Bitácora – Semana 1

## Tema de la semana
Introducción a la mecatrónica.

## Actividades realizadas
Conocimiento general de los objetivos de la materia y la situación problema.
Introducción a las áreas de estudio de la mecatrónica y orientación profesional.
Configuración de GitHub: creación del repositorio y estructuración de carpetas por semanas para mantener el orden.
Primer acercamiento al software de simulación LOGO! Soft Comfort.

## Decisiones de ingeniería
| Decisión | Alternativas | Justificación |
| :--- | :--- | :--- |
| **Estructura del repositorio** | Una sola carpeta general vs. carpetas divididas por semanas. | Se eligió la división semanal para facilitar la trazabilidad del proyecto y permitir una auditoría rápida de los avances técnicos. |
| **Uso de GitHub para documentación** | Carpetas locales. | La materia y profesor suguirieron GitHub para implementar un control de versiones real, permitiendo que todo el equipo trabaje sobre los mismos archivos. |
| **Enfoque de aprendizaje en LOGO!** | Programación en texto vs. Diagramas de funciones. | Se inició con diagramas de funciones para visualizar la lógica de control de forma gráfica antes de escalar a sistemas más complejos. |

## Problema técnico encontrado
Durante el proceso de aprendizaje para utilizar la plataforma de GitHub, tuvimos varios problemas debido a que en el equipo no estábamos familiarizados con el sistema ni con su flujo de trabajo. Al momento de crear las carpetas donde estaremos documentando nuestro progreso, surgieron complicaciones de organización ya que los archivos no se visualizaban como queríamos o simplemente no encontrábamos la manera correcta de estructurarlos inicialmente.

## Solución aplicada
Consultamos a compañeros que conocian más el uso de la plataforma y resolvimos dudas directamente con el profesor, quien estuvo al pendiente de que estuviéramos trabajando bajo las indicaciones correctas. Gracias a esto, logramos obtener un conocimiento más sólido sobre el manejo de repositorios, aunque seguimos en proceso de aprendizaje para dominar la herramienta.

## Conexión con el curso
Se aplicó el concepto de documentar todo nuestro proceso ya que en ingeniería, no basta con que el sistema funcione, es crucial que el proceso esté registrado de manera organizada y profesional. Al utilizar GitHub, alineamos el trabajo del curso con estándares de la industria, asegurando que el desarrollo del proyecto sea transparente y escalable conforme avancen las semanas.
